# あしあと仲人©️ APP_MAP v0.4I

## 0. この街にないもの

この街には、以下のものが存在しません。実装・文言ともに今後も原則追加しません。

- マッチング成立
- DM
- 即レス
- 人気ランキング
- おすすめ相手
- 条件検索
- オンライン表示

## 1. 概要

あしあと仲人©️は、効率重視のマッチングアプリでも、バズ重視のSNSでもありません。

「言葉の文脈から人となりを滲ませるコミュニティSNSの静的UI模型」です。

人をスペックで検索するのではなく、表札街を歩き、部屋の立ち話を読み、気になったら肩書きカードを見る、という順番で人となりに近づく設計になっています。ログイン・保存・通信を伴わない静的UI模型であり、実際の出会いや接続機能は持ちません。

## 2. コア思想

- 人を探す前に部屋を見る
- 部屋の空気を読む
- 立ち話から人となりを見る
- 名刺を見るように肩書きカードを見る
- 接触前に足あとや合図で減圧する
- 急がない
- 即時性を求めない
- 狩猟型ではなく散策型

## 3. 街の法律：UIで原則使わない言葉

以下の言葉は、UI文言では原則使いません。

- マッチング / マッチ
- 申請 / リクエスト
- 検索 / 絞り込み / 探す
- DM / チャット / 即レス / 送信
- 人気 / ランキング / トレンド
- アピール / いいね
- オンライン / アクティブ
- 男女比 / おすすめ相手

例外：説明文の中で「これはマッチングアプリではありません」のように、他ジャンルとの違いを説明するための比較表現として使う場合のみ許容します。UI操作の名称（ボタン文言・見出し）としては使いません。

## 4. 言葉の境界線：世界観語と機能語

- UIに表示する言葉（見出し・ボタン文言・案内文）は世界観語を使う
- 内部コード名（関数名・id・class・データ構造）は機能語のまま固定する
- 世界観語を内部ロジックの変数名・関数名・データキーに持ち込まない
- 安全、設定、エラー、不利益の告知にあたる文言は世界観語で曖昧にせず、機能語で明瞭に書く
- 世界観演出を優先しすぎて、ユーザーの安全確保や操作理解を妨げてはならない

対応例：

| UI（世界観語） | 内部（機能語） |
|---|---|
| 表札街 | screen-communities |
| 今日の名札 | screen-my-tags |
| 最近の立ち話 | threads / posts |
| 肩書きカード | screen-profile / openProfile() |
| そっと足あとを残す | showFootprint() |

## 5. 地名と看板：画面対応表

| 世界観上の呼び名 | 実体 |
|---|---|
| 受付 | 入口のご案内（screen-invite） |
| 名札選び | 今日の名札（screen-my-tags） |
| 部屋一覧 | 表札街（コミュニティ一覧）（screen-communities） |

| 住人一覧 | この部屋の住人（screen-residents） |

| 足あと / 合図 | 接触前の減圧装置（screen-footprint / screen-knock） |

## 6. 各画面の役割

**screen-top**
街の入口。急がないSNS模型であることを伝える。

**screen-invite**
入口のご案内。顔写真なし、即DMなし、立ち話から人となりを見ることを伝える。

**screen-my-tags**
今日の名札をえらぶ。タグは条件検索ではなく、表札街を歩くための小さな名札。

**screen-communities**
表札街。部屋名、タグ、中から聞こえる声、見え方、住人の気配を見る場所。

**screen-community-detail**
部屋の玄関。この部屋について、できること、最近の立ち話を見る場所。

**screen-residents**
この部屋の住人。人カタログではなく、部屋にいる住人の名札を見る場所。

**screen-thread-posts**
最近の立ち話。発言本文を主役にし、気になる人の肩書きカードへ進む場所。

**screen-profile**
肩書きカード。人となりを読む名刺。スペック表ではない。

**screen-footprint / screen-knock**
接触前の減圧装置。そっと足あとを残し、合図をえらぶ場所。

**screen-auntie-comment**
おばちゃんのひとこと。言葉を整え、接触前の圧を下げる場所。

## 7. 内部コード方針

- 関数名は変えない
- id名は変えない
- class名は変えない
- データ構造（communities / profiles / threads / posts / selectedMyTags）は変えない
- 世界観語はUI文言だけに使う
- openCommunity() や showScreen() などの内部関数名を世界観語にリネームしない
- Claudeや他のAIに修正させる場合、全面作り直しではなく最小差分を原則にする
- 1回の修正は「対象画面・対象関数・変更内容」を明確に絞って依頼する
